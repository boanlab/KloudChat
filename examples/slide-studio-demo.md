`local/qwen3.5:122b` 가 전문 발표 디자이너 프롬프트(전략 Architect → 제작 Designer 2단계)로 **자체완결형 HTML 발표자료**(인라인 CSS/JS/SVG)를 `:::artifact{type="text/html"}` 로 직접 저작한다 — 도구 없음. 우측 패널에 16:9 슬라이드로 렌더되고 키보드(←/→)/하단 버튼으로 넘긴다.

발표 **유형을 자동 판별**해 톤·밀도·구조를 바꾼다: 피칭(설득·임팩트·후크), 보고(BLUF 결론 우선·데이터), 강의(단계적·정의/예시/요약). 매 슬라이드는 **주장형 헤드라인**(라벨 아님)으로 한 메시지를 담고, 카드·표·플로우 다이어그램·SVG 차트·stat 을 섞어 표지·간지·마무리까지 구성한다.

> ⏱ 122b 가 덱 전체를 통째로 저작하므로 **~10분** 걸린다. 데모 시 "지금 돌리면 몇 분
> 걸린다"고 미리 안내. 아티팩트 토글이 켜져 있어야 우측 패널에 렌더된다.

## 1. 투자 피칭 (설득형)

- `AI 기반 재고관리 SaaS 시리즈A 투자 피칭 발표자료 만들어줘. 문제·시장·솔루션·트랙션·경쟁우위·팀·투자제안 흐름으로.`
- `개발자 컨퍼런스 키노트용 발표자료: "왜 엣지 AI 인가" 주제로, 후크 강한 표지와 데모 시나리오, 클로징 CTA 포함.`

## 2. 업무 보고 (BLUF·데이터형)

- `2025년 클라우드 인프라 운영 결과 보고 발표자료. 비용 추이, 서비스별 가용성 비교, 장애 대응 프로세스, 내년 개선 계획 포함. 결론부터.`
- `이번 분기 마케팅 성과 보고: 핵심 지표(전환율·CAC·ROAS) 표와 추세 차트, 잘된 점/문제점, 다음 분기 액션을 의사결정 위주로.`

## 3. 강의 / 교육 (단계적·충실형)

- `제로 트러스트 보안 아키텍처 교육 발표자료. 개념·핵심 원칙·구성요소·도입 단계·기존 경계보안과의 비교를 충실하게.`
- `비전공자 대상 "트랜스포머와 LLM 기초" 강의 슬라이드. 정의 → 동기 → 어텐션 개념 → 예시 → 정리 순으로, 쉬운 비유와 도식 위주.`

## 4. 주제만 주고 빠르게

- `'딥러닝 기초' 발표자료 10장으로 만들어줘.`
- `우리 회사 신입 온보딩용 "개발 문화 소개" 슬라이드 만들어줘.` → 정보가 부족하면 유형/청중/분량을 1회 간결히 묻고, 아니면 합리적 가정으로 끝까지 완성한다.

## 5. 비주얼 강조 요청

- `시장 규모(TAM/SAM/SOM)와 3년 매출 전망을 차트로 보여주는 발표자료. 표·플로우·SVG 차트를 적극 활용하고 글은 핵심만.`
- `머신러닝 파이프라인을 단계별 플로우 다이어그램으로, 알고리즘 비교는 표로 정리한 강의 슬라이드.`

## 6. 수정 / 이어가기

- *(생성 후)* `톤을 더 차분한 블루-그레이 보고서 스타일로 바꿔줘.`
- *(생성 후)* `3번 슬라이드 내용을 더 충실하게, 근거 수치를 추가해줘.` → 전체 일관성(팔레트·톤·내러티브)을 유지한 채 반영한다.

## 데모 팁

- **유형을 말해주면** 톤·밀도가 더 정확해진다("피칭/보고/강의"). 안 줘도 자동 판별한다.
- 결과는 **단일 HTML 아티팩트** — 우측 패널에서 ←/→ 로 넘기고, `.html` 로 저장하면 브라우저에서 그대로 발표 가능(외부 의존성 없음). PowerPoint(.pptx) 산출은 현재 경로엔 없다.
- 차트/표는 **실제 수치**가 있어야 의미 있다 — 데이터를 주면 SVG 차트·표로, 없으면 지어내지 않고 구조(카드/플로우)로 푼다.
- "표·다이어그램을 적극 활용"이라고 하면 카드 일변도 대신 표/플로우/차트가 고루 섞인다.
- 122b 라 한 덱에 ~10분. 데모는 미리 한 번 생성해 두거나 생성 시간을 안내하는 게 좋다.
