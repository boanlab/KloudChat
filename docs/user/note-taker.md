# Note Taker 활용 가이드

> 회의·강의·인터뷰 **녹음을 받아 깔끔한 문서로 정리**하는 에이전트 (`local/gemma-4-26b`).
> 음성 파일을 올리면 자동 전사(한국어·영어 자동 감지) → 회의록·강의노트·보고서로 구조화한다.
>
> 일상 작업·웹 검색은 [Super Agent](super-agent.md), 발표자료는 [Slide Studio](slide-studio.md) 로.

## 도구 구성

전사는 **LibreChat 내장 STT** 가 업로드 시점에 처리한다 — Note Taker 가 별도 전사 도구를 호출하지 않는다. 오디오를 첨부 메뉴의 **「텍스트로 업로드」** 로 올리면 시스템이 whisper(STT)로 전사해 **전사 원문을 텍스트 첨부로** 넣어주고, gemma 가 그 전사문을 읽어 정리한다.

**LibreChat 빌트인**

| 도구 | 하는 일 |
|---|---|
| 내장 STT (whisper) | 업로드 시 오디오 자동 전사 (「텍스트로 업로드」 경로) |
| `file_search` | 회의 참고 PDF 등 **전사 외 문서 대조**용 보조 RAG |

> 추가 MCP 없음. 전사 백엔드(whisper)는 **GPU 전용**(OR 폴백 없음) — whisper 백엔드가 있는 배포에서만 동작한다.

### ⚠️ 반드시 「텍스트로 업로드」로

- 첨부 메뉴에서 **「텍스트로 업로드」** 를 골라야 전사문이 컨텍스트로 들어온다.
- **「파일검색용 업로드」는 오디오를 처리하지 못한다** — 이걸로 올리면 전사가 안 되니, Note Taker 가 "다시 「텍스트로 업로드」로 올려달라"고 안내한다.
- 지원 포맷: mp3 / m4a / wav / ogg / flac 등. 긴 회의(1시간+)도 업로드 가능(타임아웃 없음).

## 1. 회의록 정리

*(회의 녹음을 「텍스트로 업로드」로 첨부 후)*

- `녹음 파일을 첨부해 「텍스트로 업로드」로 올린 뒤 회의록으로 정리해줘.`
- `이 회의 녹음을 회의록으로 정리해줘 — 안건·주요 논의·결정사항·액션 아이템(담당/기한) 위주로.`

→ 제목/일시 · 참석자(전사에서 식별되면) · 안건 · 주요 논의 · **결정사항** · **액션 아이템** · 다음 일정 구조로 정리.

## 2. 강의노트

*(강의 녹음 첨부 후)*

- `강의 녹음으로 강의노트 만들어줘.`
- `이 강의를 복습용 노트로 — 핵심 개념(정의·예시), 흐름 요약, 키워드, 복습 포인트로 정리해줘.`

## 3. 보고서 형식

*(인터뷰·세미나 녹음 첨부 후)*

- `이 인터뷰 녹음을 보고서로 정리해줘 — 개요·배경·주요 내용·결론·시사점 순으로.`

## 4. 출력 형식 지정

- `회의록인데, 액션 아이템만 담당자별 표로 따로 빼줘.`
- `결정사항과 미결 안건을 구분해서 정리해줘.`

## 5. 참고 문서 대조 (file_search 연계)

*(녹음 + 관련 PDF/문서를 함께 첨부)*

- `이 회의록을 첨부한 기획안 PDF 와 대조해서, 합의된 변경사항이 문서에 반영됐는지 짚어줘.`

## 활용 팁

- 산출물(회의록/강의노트/보고서)을 **미리 지정**하면 바로 그 구조로 정리한다. 안 주면 회의록 기본 — 어떤 형식이 좋을지 짧게 되물을 수 있다.
- **전사 원문에 근거해서만** 쓴다 — 없는 내용을 지어내지 않고, 안 들리는 부분은 `[불명확]` 으로 표시한다.
- 결정·수치·고유명사·날짜는 보존하고, 화자 구분은 전사에 화자 라벨이 있을 때만 반영한다.
- 출력 언어는 사용자 언어에 맞춘다(기본 한국어). 영어 회의도 한국어 회의록으로 받을 수 있다.
